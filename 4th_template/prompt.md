Ты конвертируешь транскрипт в JSON для скрипта субтитров After Effects.

ФОРМАТ ОТВЕТА: твой ответ должен начинаться с символа { и заканчиваться символом }
Никакого текста до или после. Никаких markdown блоков. Никаких пояснений. Только сырой JSON.

ВХОДНЫЕ ДАННЫЕ: транскрипт с тайм-кодами (приложен ниже).

ВЫХОДНОЙ ФОРМАТ:
{
  "word_timings": [
    { "word": "это", "start": 0.000, "end": 0.200, "focus": false },
    { "word": "мощный", "start": 0.200, "end": 0.500, "focus": true },
    { "word": "удар", "start": 0.500, "end": 0.800, "focus": true },
    { "word": "по", "start": 0.800, "end": 0.950, "focus": false },
    { "word": "системе", "start": 0.950, "end": 1.500, "focus": true }
  ],
  "subtitles": [
    { "text": "ЭТО МОЩНЫЙ УДАР", "in": 0.000, "out": 0.800 },
    { "text": "ПО СИСТЕМЕ", "in": 0.800, "out": 1.500 }
  ]
}

ПРАВИЛА для word_timings:
- Каждое слово отдельной записью с точным start и end в секундах (3 знака после точки)
- "focus": true — эмоционально сильные, ключевые слова (минимум 1 на каждые 2 субтитра). Эти слова окрашиваются красным цветом в видео
- "focus": false — все остальные слова
- Знаки препинания убирай из слова, но учитывай при расчёте позиции

ПРАВИЛА для subtitles:
- Группируй слова в субтитры по 2-4 слова (максимум 5, только если все слова короткие)
- Максимум 2 строки при ширине блока ~900px, шрифт Montserrat-BoldItalic 60px, tracking -25
- Если фраза длиннее ~15 символов — она автоматически перенесётся на вторую строку; учитывай это при группировке, чтобы не допустить 3-ю строку
- "in" = start первого слова группы
- "out" = end последнего слова группы (без padding)
- Не разрывай смысловые группы (существительное + прилагательное, глагол + наречие)
- Между субтитрами gap минимум 0.042 секунды

ПРАВИЛО КОНСИСТЕНТНОСТИ FOCUS-СЛОВ:
- Каждое слово с "focus": true в word_timings должно точно совпадать (без учёта регистра и пунктуации) с одним из слов в соответствующем subtitle.text
